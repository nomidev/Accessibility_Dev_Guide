1.적절한 대체 텍스트 제공

이미지 사용시 반드시 alt태그를 사용해야 하며 alt태그의 내용은 이미지에 있는 텍스트와 같은 내용을 삽입한다.
만약 이미지에 텍스트가 없거나 의미없는 이미지(불릿이나 아이콘등)일 경우에는 빈값으로 넣어준다.

만약 이미지 내용이 많은 경우 IR기법이나 longdesc로 대체해도 됨

2.자막제공

멀티미디어 컨텐츠에는 자막, 원고 또는 수화를 제공해야한다.

(참조자료 Naver 널리 Daum 다룸)

3.색에 무관한 컨텐츠 인식

컨텐츠는 색에 관계없이 인식될 수 있어야 한다.
페이징 같은 경우 색상만으로 구분하기 보다는 폰트 굵기/밑줄과 같은 것으로 구분해주는 것이 인식이 더 용이하다.
또 그래프나 차트의 경우도 색상만으로 구별하기 보다는 색이나 모양 패턴으로 구분주는 것이 좋다.

7.키보드 사용 보장

모든 기능은 키보드만으로도 사용할 수 있어야 한다.

  • 마우스로 작동되는 동작은 키보드로도 작동되야 한다.
  • <a>에 href속성이 없어 키보드 포커스가 접근할 수 없는 경우
  • 레이어 팝업의 경우 팝업 내에서도 키보드 포커스 이동이 논리적으로 사용 가능해야 하며 팝업이 닫힐경우 팝업이 열리기 전 위치로 포커스를 이동시켜야 한다.

8.초점 이동

키보드에 의한 초점은 논리적으로 이동해야 하며, 시각적으로 구별할 수 있어야 한다.

  • 초점을 막지 않아야 함
  • 특정 영역(컨트롤, 사용자 입력 등)이 위치 지정 도구(마우스)나 키보드 조작을 통해 초점(선택/클릭)을 받았을 때, 해당 영역을 시각적으로 인지할 수 있도록 초점을 막지 말아야 함
  • 논리적인 순서
  • 사용자가 키보드를 이용하여 초점을 이동할 때, 해당 초점이 논리적인 순서에 따라 진행해야 함 (일반적으로, 좌측 상단에서 우측 하단 방향으로)
  • 인터렉션 개체들의 초점을 강제로 제거하지 않음
  • tabindex 사용하지 않는 것을 권장
  • tabindex를 권장하지는 않지만 레이어팝업의 경우 레이어팝업으로 포커스 이동이 있을 수 있게때문에 tabindex사용(단 레이어팝업이 닫혔을 경우 포커스 이동 전으로 가야함)

16.사용자 요구에 따른 실행

사용자가 의도하지 않은 기능(새 창, 초점 변화 등)은 실행되지 않아야 한다.

  • 실행은 onclick, enter 로 가능하게 적용한다.
  • 새창 및 팝업창의 경우 a 요소에 target="_blank" 로 지정한다.
  • 웹 사이트 로딩시 자동으로 실행되는 콘텐츠(팝업창, 레이어 팝업, 배경음 포함) 을 사용하지 않는다.
  • input에 포커스만으로 새창이나 페이지 이동이 되서는 안된다.

target="blank"를 사용한 경우 title="새창열림" 생략가능

18.표의 구성

표는 이해하기 쉽게 구성해야 한다.
복잡한 데이터를 표로 제공할 경우, 시각장애인 등도 이해할 수 있도록 표의 이해를 돕기 위한 내용 및 구조에 대한 정보를 제공해야 한다.

  • summary 속성 및 <caption> 요소 유무 체크
  • summary 속성 값과 <caption> 요소 값의 중복 여부 체크
  • <th> 요소 유무 체크
  • scope 속성값이 적절한지 체크

20.오류정정

사용자 입력 오류가 발생했을 시, 사용자에게 알려주고 수정이 필요한 곳에 포커스가 이동

21.마크업 오류 방지

마크업 언어의 요소는 열고 닫음, 중첩 관계 및 속성 선언에 오류가 없어야 한다.

  • 태그의 열고 닫음 체크
  • 태그의 중첩이 올 바른지 체크
  • 한 요소에 중복된 속성 선언 여부 체크
  • 한 페이지에 중복된 id 속성값 선언 여부 체크
  • 인라인 요소안에 블록요소 포함 여부 체크(대표적 인라인 요소 span)
